# v77 canonical preview / published PDF parity

This version keeps PDF preview and published PDF on the same backend rendering path.

Changes:

- `/api/publish-pdf` now generates both preview-only PDFs and published PDFs through the same Cloudflare Browser Rendering HTML template and CSS.
- The canonical PDF output includes a centered footer page number on every page: `Page X`.
- The canonical PDF output includes the running header text `AIBioTʀχiv AI BIOTHEORY ARCHIVE` through the backend template instead of relying on each user browser's print engine.
- The frontend PDF preview button continues to call the backend preview mode, so authors and admins see the same canonical PDF style that publishing uses.
- The publish route continues to write the final PDF and HTML snapshot only when `previewOnly` is false.

Operational note: after deploying, generate a new PDF preview, then publish from the same saved layout. Existing PDFs already stored in R2 must be regenerated by returning the manuscript to Accepted and publishing again.
