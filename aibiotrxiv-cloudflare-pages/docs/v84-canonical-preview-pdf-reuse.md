# v84 canonical preview PDF reuse

This version fixes PDF preview / published PDF mismatch by making the published PDF reuse the exact canonical preview PDF object generated by Cloudflare Browser Rendering.

## Changes

- PDF preview calls `/api/publish-pdf` with `previewOnly=true` and `storeCanonicalPreview=true`.
- The backend stores the generated canonical preview PDF and HTML snapshot in R2 under `canonical-preview/<articleId>/<version>/...`.
- The preview response returns R2 keys in response headers.
- Admin and member layout workspaces remember those keys in their local/D1 paper records.
- Publish is blocked until PDF preview has been generated.
- Publish sends the remembered canonical preview R2 keys to `/api/publish-pdf`.
- `/api/publish-pdf` copies the exact preview PDF bytes into the published R2 path instead of rendering a second PDF.

This ensures the PDF that was previewed is the PDF that becomes public.
